# Project Overview

This project supports two main workflows:
1. Direct fine-tuning
2. TAPT (Task-Adaptive Pretraining) + fine-tuning

Below is a description of each file and its role in the pipeline.

---

## File Descriptions

### `main.py`
- The main training script for **fine-tuning**.
- Loads the pretrained (or TAPT-trained) model.
- Handles training loop, validation, and saving the best model.

---

### `evaluate.py`
- Used to **evaluate** the trained model.
- Loads the saved checkpoint from `main.py`.
- Outputs metrics such as accuracy, F1-score, etc.

---

### `clean_text.py`
- Handles **text preprocessing**.
- Removes or normalizes noisy elements such as:
  - Usernames
  - URLs
  - Preserving the emoticons/emojis
- This script is automatically used in other parts of the pipeline.

---

### `mlm_generator.py`
- Prepares data for **Masked Language Modeling (MLM)**.
- Extracts raw text from the dataset into a `.txt` file.
- Applies preprocessing using `clean_text.py`.
- This output is used for TAPT.

---

### `special_token_added.py`
- Adds **custom special tokens** to the tokenizer and model.
- Example tokens:
  - `<URL>`
  - `<USER>`
- Helps the model better understand structured text patterns.
- Recommended before running TAPT.

---

### `run_mlm.py`
- Script for **Task-Adaptive Pretraining (TAPT)** using MLM.
- Typically adapted from HuggingFace’s official training script.
- Trains the model on domain-specific text generated by `mlm_generator.py`.
- Produces a new checkpoint for downstream fine-tuning.

---

## Workflow Summary

### Option 1: Direct Fine-tuning
1. Run `main.py`
2. Run `evaluate.py`

---

### Option 2: TAPT + Fine-tuning
1. Run `mlm_generator.py` → generate training corpus  
2. (Optional) Run `special_token_added.py` → improve tokenization  
3. Run `run_mlm.py` → perform TAPT  
4. Update model path  
5. Run `main.py` → fine-tune  
6. Run `evaluate.py` → evaluate  

---

## Notes
- Ensure `requirements.txt` is installed before running any scripts.
- After TAPT, always update the model path in `main.py`.
- If `main.py` includes steps to add special tokens, remove them if you have already run `special_token_added.py` (or handled token addition elsewhere).
